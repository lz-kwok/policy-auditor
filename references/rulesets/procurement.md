# 📋 采购与招标管理制度审计细则 (Procurement Ruleset)

本细则适用于对企业**采购管理办法、招投标管理制度、供应商管理细则、比质比价管理办法、付款申请审批流程**等制度的专业化审计。

---

## 🔍 一、 核心内控红线检查点 (Core Control Checkpoints)

在审计采购与招投标相关制度时，Agent 必须强制扫描以下内控红线的合规性：

### 1. 职责分离 (SoD)
* **需求与采购分离**：提报采购需求（计划）的部门/岗位，不得直接负责采购寻源和商务谈判。
* **谈判与定标分离**：商务谈判或比价小组成员，不得单方面拥有最终定标决策权。
* **验收与付款分离**：负责现场收货、入库或工程节点验收的岗位，不得兼任付款申请审批或付款执行岗位。

### 2. 招投标准入红线
* **起标金额硬阈值**：制度中必须明确定标额度边界。例如：单项或年度累计预计采购额超 X 万元必须公开招标，超 Y 万元必须邀请招标，严禁无红线模糊授权。
* **拆标检测**：制度中必须明文禁止“化整为零”将应招标项目拆分为小额单一来源或询比价采购的行为。
* **评标小组组建**：评标小组成员应多于 3 人以上的奇数，且必须包含法务、财务或独立第三方评审人员，严禁业务部门独立定标。

### 3. 供应商管理
* **准入与交易分离**：供应商准入资格审核（入库评估）必须由独立的评估委员会或跨部门联合评审，业务寻源员不得自行直接准入。
* **黑名单机制**：必须有明确的供应商违约惩戒与黑名单退出机制。

## 🚨 二、 典型漏洞风险库 (Typical Risks)

### 1. 紧急采购与单一来源采购滥用
* **风险等级**：`P1` (高风险)
* **漏洞剖析**：制度中若允许“因紧急情况或特殊要求可直接采用单一来源采购”，但未严格限定“紧急情况”的定义、多级审批链，以及不可替代性技术证明，会导致采购人员故意拖延提报时间，从而人为制造“紧急情况”以规避招投标。
* **审计要求**：审查是否有“紧急采购”和“单一来源”的前置技术论证、总经理级/委员会会签审批规定。
* **数字化硬校验建议**：在 ERP/SRM 系统中，单一来源采购必须附有经审批的技术唯一性论证附件，否则系统不允许提交采购订单。

### 2. “拆单”规避招标漏洞
* **风险等级**：`P1` (高风险)
* **漏洞剖析**：制度未禁止在同一项目、同一供应商下拆分合同。
* **审计要求**：审查是否有“禁止将同一采购项目或同一预算项下的物料拆分成多个小额订单/合同以规避起标额度”的惩戒与监控规定。
* **数字化硬校验建议**：系统对相同项目/相同供应商在 30 天内累计提报金额进行监控，累计超招投标阈值时系统强制锁定并路由至审计部门预审。

### 3. 串标围标防范漏洞
* **风险等级**：`P0` (违法违规)
* **漏洞剖析**：招投标流程流于形式，制度中缺少对“关联投标方”的限制，或者对投标保证金来源、投标文件制作机器 IP 码/物理介质的一致性检查缺失。
* **审计要求**：审查是否规定了“投标文件雷同性审查”（如电子标书 MAC 地址、IP 地址一致性拦截）和投标方关联关系审查规则。
* **数字化硬校验建议**：在招投标系统（SRM）中，系统自动对比各投标单位标书的机器特征码（MAC地址）和 IP 地址，若一致则自动报警并废标。

### 4. 预付款比例超限与结算风险
* **风险等级**：`P1` (高风险)
* **漏洞剖析**：制度允许无担保、大比例支付预付款，导致资金占用甚至供应商卷款跑路风险。
* **审计要求**：审查是否有预付款上限要求（如不得超 30%），超限是否要求提供等额“银行履约保函”。
* **数字化硬校验建议**：ERP 支付申请中，若勾选“预付款”且金额超合同总价 30%，系统自动拦截并提示上传“履约保函”附件，否则无法发起付款审批。
